Gnome fails at pen input

Last updated on November 1, 2016 by Jean-Philippe Bernardy
Tags:

this bug is a total showstopper: https://bugs.launchpad.net/ubuntu/+source/gnome-shell/+bug/1600371

Unity does not suffer from the bug, but is unusable anyway, because it grabs and interprets all 3+ point touches. Those happen with regularity in a note-taking application due to involuntary resting of the hand on the screen.

In any case, I do not think that it is acceptable for the desktop environment to ‘reserve’ so many touch gestures: they should be available to apps. At the very least I need to be able to configure which gestures GNOME recognizes. (For me only the left edge should be reserved for GNOME, as I’m unlikely to touch it when I’m writing).

Again, I agree that GNOME is great for pen-less convertibles with no touch-aware applications. It’s a shame that it breaks down so completely, and so close to the winning line.
